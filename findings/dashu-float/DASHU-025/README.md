# DASHU-025: powi discards directed range endpoints and panics in the out-of-range reciprocal path

Status: confirmed on the locked baseline. Confidence: high. Classification: `incorrect-result`.

Contract: `backend_conformance`. Owner: `backend`. Masked by adapter: `true`.

Latest release check: Reproduces with dashu-float 0.5.0 and current Dashu master 40f465b62e5d8f4198efc43871e3ce601d03dc93 in debug and release builds. No matching upstream issue or fix was found when verified on 2026-07-20.

## Summary

For exact powers beyond FBig's exponent range, FBig::powi converts Context overflow and underflow errors to the same infinity or zero regardless of directed rounding. For example, Down 2^(isize::MAX+1) returns positive infinity instead of the largest finite precision-1 FBig. Negative exponents recurse through a positive power; once that intermediate saturates to infinity, the reciprocal attempts finite arithmetic with infinity and panics.

## Impact

The convenience API returns endpoints on the wrong side of the exact value, invalidating directed bounds, and some equivalent reciprocal formulations panic instead of returning a directed range endpoint. The opendp-num primitive adapter masks these cases with structural primitive-range classification.

## Tested baseline

`dashu 0.5.0`, `dashu-float 0.5.0`, `dashu-int 0.5.0`, `dashu-ratio 0.5.0`, `malachite 0.9.2`, `rug 1.30.0`.

## Deduplication rationale

Separate from DASHU-024, whose exact result is still representable and is lost through endpoint-equality errors. DASHU-025 covers genuinely out-of-representation-range results: rounding-mode-independent saturation and the reciprocal panic are two manifestations of the same powi range-error handling path.
